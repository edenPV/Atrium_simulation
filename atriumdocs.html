What is Atrium?

A 3-tranche principal-first primitive for on-chain RWA and yield-bearing stablecoins, targeting institutional LPs. Senior principal is denominated in USD rather than the staked underlying, with risk-premium structurally routed down to Mezzanine and Junior.

Losses absorb in order Junior → Mezzanine → Senior. Yield distributes in reverse: Senior receives Sr_yield_target first, residual splits between Mezzanine and Junior with Junior receiving a priority-weighted share of the Risk Premium.

Accounting model

The Accounting module is a pure state contract that computes yield distribution and loss waterfall at every updateAccounting call. All formulas are conservation-enforced: Y_sr + Y_mz + Y_jr = netGain at every settlement.

Both RP_nominal and RP_delivered are tracked. When floor_sr is active, Senior receives the floor and the actual RP flowing to sub is reduced below nominal. The MAX(0, …) construction in RP_delivered handles this automatically, no separate floor-on / floor-off branch.

Risk Premium curve

Senior pays a risk premium to the subordinate pool. RP_nominal scales with Senior's concentration: when Senior is a large share of the pool (thin sub below it), it pays more. When sub is thick, RP tapers. This creates a self-correcting incentive that attracts sub capital when protection is thin.

TVLRatio_sr = TVL_sr / pool
RP_nominal  = x_sr + y_sr × TVLRatio_sr ^ k_sr
ParameterDefaultDesign intent
x_sr (baseline)15%Minimum premium Senior always pays
y_sr (variable)20%Sensitivity to Senior concentration
k_sr (curvature)0.3Concave curve. Rapid response at low TVLRatio_sr, plateau at high.

RP distribution weights

RP_delivered is split between Mezzanine and Junior using priority-weighted shares. Junior's weight is multiplied by α to compensate for first-loss exposure. Mezzanine receives its proportional share without the multiplier.

share_jr = (α × TVL_jr) / (α × TVL_jr + TVL_mz)
share_mz = TVL_mz / (α × TVL_jr + TVL_mz)

// Conservation: share_jr + share_mz = 1
// With α = 2.0 and TVL_mz = TVL_jr → share_jr = 66.7%, share_mz = 33.3%
// With α = 1.0 (no priority) → shares match TVL proportions

Reserve cut

reserveCut = strategyGain × reserveBps / 10000   // only when strategyGain > 0
netGain    = strategyGain − reserveCut
BaseAPY    = GrossBaseAPY × (1 − reserveBps / 10000)

// Reserve cut applies only on positive gains.
// Default reserveBps = 500 (5%).

Case classification

Distribution path is determined by comparing netGain against Senior's yield target:

Case 1: netGain ≥ Sr_yield_target
Case 2: netGain <  Sr_yield_target
Case 1: Sr target met

NetGain is sufficient to cover Senior's target. The subordinate pool receives the residual. In the ideal case, Sub_APY = BaseAPY. When floor_sr is active and RP_delivered drops to zero, Sub_APY can fall below BaseAPY, the system is in the yield-compressed sub-case.

Step 1, Senior
   Y_sr = Sr_yield_target

Step 2, Sub pool
   Sub_yield_total = netGain − Sr_yield_target − RP_delivered
   Sub_APY         = Sub_yield_total / sub

Step 3, Mezzanine and Junior
   Y_mz = Sub_APY × TVL_mz + share_mz × RP_delivered
   Y_jr = Sub_APY × TVL_jr + share_jr × RP_delivered

// Ideal case  : Sub_APY = BaseAPY, hierarchy APY_sr ≤ BaseAPY < APY_mz < APY_jr
// Compressed  : Sub_APY < BaseAPY; when RP_delivered = 0 → APY_mz = APY_jr = Sub_APY

Conservation: Y_sr + Y_mz + Y_jr = netGain
Case 2: Sr target not met

NetGain is insufficient to cover Senior's target. The sub pool absorbs the deficit through the loss waterfall: Junior yield first, then Junior principal, then Mezzanine yield, then Mezzanine principal. Senior is impaired only in extreme cases where the sub pool is fully exhausted.

Step 1, Compute deficit and total absorb
   deficit      = Sr_yield_target − netGain
   total_absorb = deficit + BaseAPY × sub
   // covers (a) deficit Sr needs and (b) sub yield base

Step 2, Compute capacities
   Jr_capacity = MAX(0, TVL_jr × (1 + BaseAPY))
   Mz_capacity = MAX(0, TVL_mz × (1 + BaseAPY))
   // remaining value of tranche after BaseAPY

Step 3, Apply waterfall
   jr_absorb = MIN(total_absorb, Jr_capacity)
   mz_absorb = MIN(total_absorb − jr_absorb, Mz_capacity)
   sr_loss   = total_absorb − jr_absorb − mz_absorb  // > 0 only in extreme

Step 4, Final yields
   Y_sr = Sr_yield_target − sr_loss
   Y_mz = BaseAPY × TVL_mz − mz_absorb
   Y_jr = BaseAPY × TVL_jr − jr_absorb

Conservation: Y_sr + Y_mz + Y_jr = netGain

Coverage ratio and cooldown

CR_SR is the primary health indicator. Mezzanine and Junior share the same cooldown logic, there is no Mz-specific coverage metric because Mz is not protected by Jr with respect to yield.

StateCR_SRSeniorMezzanineJunior
Normal > 1.20× NONE (instant) NONE (instant) NONE (instant)
Monitoring 1.10–1.20× NONE (instant) SHARES_LOCK (3d) SHARES_LOCK (3d)
Warning < 1.10× NONE (instant) SHARES_LOCK (7d) SHARES_LOCK (7d)
Critical < 1.05× Mint paused SHARES_LOCK (7d) SHARES_LOCK (7d)
Atrium uses a single cooldown mechanism: SHARES_LOCK. The previous two-mode design (ASSETS_LOCK + SHARES_LOCK) is replaced by a single mode with two duration tiers gated on CR_SR. Senior withdrawal is always NONE, hardcoded invariant.

Invariants

  • INV-BALANCE: TVL_sr + TVL_mz + TVL_jr + reserve ≡ Strategy.totalAssets()
  • INV-PRIORITY: Y_sr computed before Y_mz and Y_jr, order cannot be reversed
  • INV-SENIOR-LIQUID: Senior withdrawal is always NONE, hardcoded
  • INV-LOSS-ORDER: Capital losses absorb Junior → Mezzanine → Senior, no skip, no reorder
  • INV-CONSERVATION: Y_sr + Y_mz + Y_jr = netGain, enforced in both cases
  • INV-APY-FLOOR: APY ≥ −100% for all tranches, enforced by MAX(0, …) in capacities
  • INV-RP-BOUND: RP_nominal < 100%, enforced by x_sr + y_sr ≤ 80%
